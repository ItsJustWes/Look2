phpBay Pro API Documentation

phpBay is an eBay script that queries eBay's XML service to quickly and easily incorporate eBay auction listings into your website. The script is written in php and is designed to integrate into your existing website with just a few lines of code. Most parameters are preset, so that you only need to provide a keyword(s) to get going. Of course, there are a number of properties available to fine tune the results the script provides. The script serves to obfuscate any relationship (links) to eBay, so that search engines which may target sites with numerous affiliate links to eBay, will not be able to detect the page offers eBay auction links.

Quick Start

The script uses three files to operate:

  • ebay.php - contains the ebay and xml classes that do all the work.
  • auction.php - all links to ebay auctions are base64 encoded (obfuscation) and redirected through this file and on to the eBay auction. You can rename this file to something else, if you like. There is a reference to auction.php in the core script in ebay.php that will need to be changed as well.
  • template.ebay.results.html - a simple template file that contains the structure for how each eBay listing will return. Each listing will use this format to be listed. You can modify this file to use CSS only, add your CSS class to the table and or td elements, etc. The important parts of the template are the %%variables%%. These will be replaced with actual listing data for each listing.

Additionally, there is a file called .htaccess. If your site already has a .htaccess file, simply merge the contents of the included .htaccess file into your existing one. The code performs a mod rewrite on the image file returned from the eBay XML query for each item, so that the image does not appear to be coming from eBay, but rather your own site. If you do not have an existing .htaccess file, then copy this one to your server. Another file, called example.php is included, to illustrate how simple it is to incorporate into your site and what lines of code you will need to get going.

To get started, open ebay.php to set a few parameters:

$site_url - The url to your site, including trailing forward slash and/or directory name. Wherever your these files are going to reside, is the url you should enter. Examples:

  • $site_url = "http://www.yoursite.com/"
  • $site_url = "http://www.yoursite.com/auction/"

$aff_type - You will use either Commission Junction or Media Plex. If you do not have an eBay affiliate ID, you can join either of these affiliate services and request an eBay affiliate ID. If you use Media Plex, the value will be "afmp". If you use Commission Junction, the value will be "afcj".

$ebay_pid - Your eBay affiliate ID from either Commission Junction or Media Plex. Note, this is not your eBay user ID, it is a special affiliate ID you will need to sign up for, through one of these affiliate services.

$encode - Leave this value set to 1. The affiliate links/urls are encoded with base64 so that search engines do not devalue your site for having excessive affiliate urls. To see the actual affiliate urls, set to 0 for testing only. This will allow you to see that your $ebay_pid is set correctly.

Next, if you have an existing .htaccess file on your site, open the .htaccess file that comes with phpBay and merge these two lines with your current .htaccess file:

RewriteEngine On
RewriteRule ^images/e/(.*)$ http://thumbs.ebaystatic.com/pict/$1 [R,L]

Using your FTP program, upload:

  • ebay.php
  • .htaccess
  • auction.php
  • template.ebay.results.html

to your server. Direct your browser to:

http://www.yoursite.com/example.php

Enter a product keyword to search for, such as: apple ipod

You should see a listing of auction items relating to the apple ipod display on your screen.

Merging phpBay Into an Existing PHP Based Site

Now that you have the example working, integration into your existing php based website is very simple. You'll need to include one file into your existing page along with a few lines of code:

<?
 require_once("ebay.php");
 $keywords = "apple ipod";
 # ebay category is optional.  if not used, it will default to search all categories
 $category = ""; 
 # create an instance of the ebay class
 $ebay = new ebay();
 # set the number of listings to display per page
 $ebay->eb_frpp = "10";
 # create the listings content
 $ebay->listings($keywords, $category);
 #display the listings
 echo $ebay->html;
 ?>

Let's break the lines of code down, so you can better understand what is taking place.

Line 1 - require_once("ebay.php"); - This line simply includes the ebay.php file that is needed to do all the work. It contains the phpBay code that will display auction listings on your site.

Line - 2 - $keywords = "apple ipod"; - Here we define the product keywords we want to retrieve listings for. Of course, you do not have to hard code these. For example, if you want to display dynamic results based upon what your web page is about, then you probably already dynamically create your page title. Let's say your site is about "mp3 players" and one of your page title is generated as:

$pageTitle = "mp3 Players - Apple Ipod";

Then you would simply set line 2 as:

$keywords = $pageTitle;

There are any number of ways to pass a value about your page/product to $keywords.

Line - 4 - $category = ""; - $category is optional. You do not need to specify a category to search and if you do not, phpBay will search all categories and return results. However, you may find, at times, you do not get the most relevant listings returned. In this case, you can browse eBay's Category Listings and find the most appropriate category.

For example, continuing with our theme of "apple ipod" we would visit the link above to find a specific category. Under "Consumer Electronics" exists a category called "MP3 Players & Accessories". Click this link. On the next page, in the left hand column under "Categories" there is another category called "Apple iPod, MP3 Players". On the next page, on the left hand column under "Categories" there is another category called "Apple iPod". Let's use this category.

Finding the Ebay Category

http://electronics.listings.ebay.com/Apple-iPod-MP3-Players_Apple-iPod_
W0QQfclZ1QQfsooZ2QQfsopZ2QQsacatZ67833QQsocmdZListingItemList

Note the instances of "QQ" in the url string? "QQ" is a parameter separator. Instances of "Z" are actually the equal sign "=". The category number is located just after "sacat" and the equal sign "Z" up through the ending "QQ". In this case, I've bolded and underlined the actual category number stripped from the url above: sacatZ67833QQ

The bolded/underlined number is actually the category number on eBay. So you would assign this value in your script code like:

$category = "67833";

Again, this is not necessary for the script to work. But if you really want to narrow down your product listings to come from specific categories on eBay, you have this option. Otherwise, just leave it blank.

Line 6 - $ebay = new ebay(); - This line creates an instance of the ebay class in ebay.php. In essence, it "starts things up."

Line 8 - $ebay->eb_frpp = "10"; - eb_frpp is a property for how many listings you want to return. You should set this value to between 1 and 100. Ideally, you might want to set it to 10, or 20. Whatever the value is, is how many auction product listings will be returned and displayed.

Line 10 - $ebay->listings($keywords, $category); - This line calls the listings() function in ebay.php that does all the work. It takes the two parameters, $keywords and $ category (remember, $category can be null "" and is not necessary, unless you want to use it), sets some initial values, performs a query on eBay's XML service and returns your auction listings in XML format. It further extracts this data and begins this data with the layout in template.ebay.results.html for each product listing. When complete, the entire product listing is stored in the next variable, which you display into your php based page.

Line 12 - echo $ebay->html; - The final step. This is the variable that contains the auction product listings and you can place it in your page where you want the listings to show.

In summary, you really only need a few lines of php code in your existing site to pull in eBay auction items:

<?
 require_once("ebay.php");
 $keywords = "apple ipod";
 $category = ""; 
 $ebay = new ebay();
 $ebay->eb_frpp = "10";
 $ebay->listings($keywords, $category);
 echo $ebay->html;
 ?>

phpBay Properties

phpBay has a number of properties that can be set, prior to calling 

$ebay->listings($keywords, $category);

to alter the way auction listings are returned. As you've seen above, there are really just a few lines of code to get product listings into the pages on your website, however, you can really fine tune the results returned by setting additional properties.

$ebay->eb_exclude Exclude Keywords - A list of words, separated by spaces, to exclude from the search. For example, if your search keyword is "apple ipod" and you wanted to exclude items that mention "docking station" you would use: $ebay->eb_exclude = "docking station"
$ebay->eb_saprclo Minimum Price - The minimum price of product listings to return. As an example of how this can be beneficial, let's suppose you are setting up a site for selling Dodge Vipers. You use the keywords "dodge viper" but the returned results range from hot wheels to toy cars and steering wheel covers for Dodge Vipers. You want real cars, Dodge Viper cars. Since we know that Dodge Vipers would easily go for more than a few dollars (as opposed to hot wheels), you could set this value to "4000" to fine tune the returned results. Example: $ebay->eb_saprclo = "4000". This means that only products that match your keywords and have a minimum bid value of $4000 will be returned. No currency symbol is needed and no decimal places needed.
$ebay->eb_saprchi Maximum Price - The maximum price of a product you want to display. As an opposite example of eb_saprclo above, if you wanted to display hot wheel cars about Dodge Vipers, you might use keywords "dodge viper hot wheels" however, a seller of a real Dodge Viper might include a hot wheels version of his car as well, so you can set the maximum price to "20" for $20.00 to exclude any keyword results over $20.00. Example: $ebay->eb_saprchi = "20". eb_saprchi and eb_saprclo are exclusive from each other. You do not have to use both. You could use both, to set a lower and upper limit of products to display if you want, or you can use one or the other.
$ebay->eb_sasl Specific Seller's Auctions Only - If you want to display your auctions only, or the auctions of a (or multiple) specific seller(s), use this property to pass in the seller(s) eBay seller IDs. For example, if you sell products on eBay and only want to list ALL of your products, then you would pass NULL "" as your keywords ($keywords = "";) and use: $ebay->eb_sasl = "yourEbayUserID". If you sell may products and only want to list specific ones by your username, then set your $keywords as well, otherwise, to display all, be sure to pass "" as your $keywords.
$ebay->eb_salic

Items Located in X Country - Seller has indicated that the item is located in X country. For example, if you only wanted to display products from the United States, you would use: $ebay->eb_salic = "1" The eBay country code for the US is 1. This may not always be accurate, as the seller may specify an incorrect country.

Use the following drop down list to select your country and note the number next to your country's name. This will be the number you use to assign $ebay->eb_salic :

 

$ebay->eb_saatc

Items Available to X Country - Seller has indicated that they are willing to ship to X country. For example, if you only wanted products where a seller will ship to Algeria, you would use: $ebay->eb_saatc = "6" The eBay country code for the Algeria is 6.

Use the following drop down list to select your country and note the number next to your country's name. This will be the number you use to assign $ebay->eb_saatc :

 
$ebay->eb_sacur

Display in X Currency - By default, prices are displayed in US dollar currency. You can change the way prices are displayed in different currencies. Use the following drop down list to select the currency and note the number next to your currency's name. This will be the number to assign to $ebay->eb_sacur . If you wanted to display prices in Euro, you would set $ebay->eb_sacur = "7"; Note: you do not need to set this property if you place to use US currency.

 
$ebay->eb_sascs Display "Buy it Now" Products Only - If you would prefer to display only products that have a "Buy it Now" option, set this value to 2, otherwise do not set the value at all. Example: $ebay->eb_sascs = "2";
$ebay->eb_siteId
$ebay->eb_language

$ebay->eb_fsop
$ebay->eb_fsoo

Sort Order - This is a dual property setting that might be a little confusing, but basically there are four options for sorting the returned product listings from ebay:

Time: ending soonest = fsop=1&fsoo=1
Time: newly listed = fsop=2&fsoo=2
Price: lowest first = fsop=3&fsoo=1
Price: highest first = fsop=3&fsoo=2

So, for each of the four sort orders, both of these properties must be set to appropriate values. By default, products are listing as "Time: ending soonest" meaning the that $ebay->eb_fsop is set to 1 and $ebay->eb_fsoo is set to one also. However, if you wanted your products displayed with "Price: lowest first" then you would set:

$ebay->eb_fsop = "3";
$ebay->eb_fsoo = "1";

$ebay->eb_sid

Use SID Tracking - If you are using SID tracking with your Ebay auction listings, you can very easily implement this into phpBay. Wherever your SID tracking code is placed, you'll need to create a global variable in php named $sid :

global $sid;

Just place the global declaration at the beginning of your php code. Then, whatever variable contains your SID value, just assign it to $sid. For example, in your Tracking code, the SID is stored in a variable named $track_code . Then at the end of your php Tracking code, you would add a line like:

$sid = $track_code;

Please don't literally use "$track_code". Your tracking code will have a variable name, so use the proper variable name.

Next, in the phpBay code you are using to display auctions, you simply need to set the following value (remember, we set these AFTER calling $ebay = new ebay();) like:

$ebay->eb_sid = $sid;

That's it! Just declare a global variable called $sid in your tracking code. Assign the variable that stores the SID value to $sid. Then add the line directly above to your phpBay code. If $sid contains a value, it will be passed along with the auction affiliate link to ebay, where you can keep track of these.

Remember, all of these properties are optional. Further, you MUST set properties PRIOR to calling:

$ebay->listings($keywords, $category);

The order is:

$ebay = new ebay(); (create the instance of the ebay class)
$ebay->eb_sid = $sid; (only if you are using SID tracking)
$ebay->eb_fsop = "3"; (set any additional properties after creating the class instance) 
$ebay->eb_fsoo = "1"; (set any additional properties after creating the class instance) 
$ebay->listings($keywords, $category);  (call the listings function to query ebay and create the data)
echo $ebay->html; (display the results)
